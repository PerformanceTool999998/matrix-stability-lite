## 面试展开点

### 1. 架构设计

**Q：为什么用 Agent-Server 分离？**

&gt; 答：测试设备通常在机房或实验室，与开发环境隔离。Agent 部署在设备侧负责采集，Server 负责解析与存储，两者通过 HTTP 通信。这种架构支持：
&gt; - 水平扩展：新增设备只需新增 Agent
&gt; - 环境隔离：Server 可部署在云端或本地，不依赖设备网络
&gt; - 容错设计：Agent 离线后本地缓存，恢复后补传

### 2. 指纹去重算法

**Q：去重算法怎么设计的？有什么局限？**

&gt; 答：Lite 版采用"堆栈特征 + MD5 哈希"：
&gt; 1. 正则提取 `at class.method(file:line)` 格式的堆栈行
&gt; 2. 取前 3 层作为特征（平衡精度与性能）
&gt; 3. MD5 哈希生成唯一指纹
&gt; 4. 数据库查重，存在则计数+1
&gt; 
&gt; **局限：** 无法处理相似堆栈（如不同内存地址的同一 bug）。如果面试深入，可以提到 Pro 版使用**参数化归一化**（忽略地址、线程 ID）解决。

### 3. 防御设计

**Q：怎么防止恶意日志攻击？**

&gt; 答：三层防护：
&gt; 1. **Pydantic 校验**：限制字段长度、类型、正则格式
&gt; 2. **内容截断**：原始日志限制 5000 字符，防止大文件 DOS
&gt; 3. **类型过滤**：只处理 ANR/Crash/Tombstone，忽略普通日志

### 4. 部署与运维

**Q：怎么部署？单机能撑多少设备？**

&gt; 答：Lite 版单机部署，SQLite 零配置，5 分钟跑起来。实测单机可支撑 10-20 台设备（取决于日志频率）。如果要扩展，可以：
&gt; - SQLite → PostgreSQL
&gt; - 单进程 → 多 Worker（Uvicorn）
&gt; - 增加 Redis 缓存热点指纹
