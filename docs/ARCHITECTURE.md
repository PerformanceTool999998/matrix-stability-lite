# MatrixStability Lite 架构说明

## 系统概述

MatrixStability Lite 是一款面向 Android 测试工程师的**学习版长稳压测系统**，采用经典的 **Agent-Server 分离架构**，帮助理解自动化测试系统的设计原理。

&gt; **Pro 版差异**：Lite 版使用简化版指纹去重算法，Pro 版使用闭源 `ClosedSourceEngine` 引擎。

## 核心模块详解

### 1. Agent 层（客户端）

| 组件 | 职责 | 技术点 |
|------|------|--------|
| 日志采集 | 模拟 ADB 日志获取 | 实际可接入 `adb logcat` |
| 类型识别 | 区分 ANR / Crash / Tombstone | 正则匹配 |
| 定时上报 | 30 秒间隔 HTTP POST | `requests` + `threading` |

**学习价值：**
- 理解客户端如何与服务端解耦
- 掌握日志采集的基本模式

#### 2.2 数据校验 (`schemas.py`)

| 技术 | 用途 | 面试可讲 |
|------|------|---------|
| Pydantic | 请求参数校验 | 替代手动 `if/else` 校验 |
| `Field` 约束 | `max_length`, `pattern` | 防御非法输入 |

#### 2.3 数据库 (`database.py` + `models.py`)

| 技术 | 用途 | 面试可讲 |
|------|------|---------|
| SQLAlchemy 2.0 | ORM 操作 | 对比 Django ORM / 原生 SQL |
| SQLite | 零配置本地数据库 | 适合原型/学习 |
| 联合查询 | 设备状态 + 崩溃统计 | 聚合查询优化 |

#### 2.4 指纹去重 (`upload.py`) ⭐ 核心

**简化版算法流程：**
原始日志 → 提取堆栈 → 取前3层 → MD5哈希 → 查重 → 新崩溃/计数+1


| 步骤 | 实现 | 面试可讲 |
|------|------|---------|
| 堆栈提取 | 正则 `at xxx.xxx(xxx.java:123)` | 如何定位关键信息 |
| 特征压缩 | 只取前 3 层 | 平衡精度与性能 |
| 哈希去重 | MD5 指纹 | 哈希冲突处理 |
| 计数合并 | 相同指纹 `occurrence_count += 1` | 数据聚合策略 |

**Pro 版差异：**
- Pro 使用**闭源指纹引擎**，支持：
  - 模糊匹配（相似堆栈归为一类）
  - 参数化去重（忽略内存地址变化）
  - 多维度聚类（堆栈 + 日志上下文）

#### 2.5 看板 (`dashboard.py`)

| 技术 | 用途 | 面试可讲 |
|------|------|---------|
| Jinja2 模板 | 服务端渲染 HTML | 对比前后端分离 |
| 聚合查询 | `func.sum`, `func.count` | SQL 优化基础 |
| 数据可视化 | 统计卡片 + 表格 | 报表设计思路 |

## 面试要点

### Q1: 为什么用 Agent-Server 分离架构？

> 答：解耦采集与解析，Agent 可部署在测试机房（靠近设备），Server 可部署在云端或本地。支持水平扩展，新增设备只需新增 Agent。

### Q2: 指纹去重算法怎么设计的？

> 答：Lite 版采用"堆栈特征 + MD5 哈希"方案：
> 1. 正则提取堆栈跟踪（`at class.method(file:line)`）
> 2. 取前 3 层作为特征（平衡精度与性能）
> 3. MD5 哈希生成唯一指纹
> 4. 数据库查重，存在则计数+1
> 
> 局限性：无法处理相似堆栈（如不同内存地址的同一 bug）。Pro 版使用闭源引擎，支持模糊匹配与参数化去重。

### Q3: 如何防止重复日志淹没系统？

> 答：三层防护：
> 1. **指纹去重**：相同崩溃只存一条，计数累加
> 2. **内容截断**：原始日志限制 5000 字符，防止大文件攻击
> 3. **类型过滤**：只处理 ANR / Crash / Tombstone，忽略普通日志

### Q4: 如果设备从 1 台扩展到 100 台，怎么优化？

> 答：
> 1. Agent 层：多线程采集，每台设备独立线程
> 2. Server 层：SQLite → PostgreSQL，连接池管理
> 3. 去重层：Redis 缓存热点指纹，减少数据库查询
> 4. 看板层：异步统计，定时任务预聚合

### Q5: 这个项目的最大难点是什么？

> 答：崩溃指纹的**相似性判断**。同一 bug 可能因内存地址不同导致堆栈差异，简单哈希会误判为不同崩溃。Pro 版的闭源引擎通过**参数化归一化**（忽略地址、线程 ID 等变量）解决此问题。
